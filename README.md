# Airclean_board_Knowledge_base

这是空气净化器控制板知识库仓库，主要用于沉淀本次板子相关的芯片手册、语音模块资料、板级引脚分配、风扇功能规格、NTC 温度数据和 Context7 知识库。

本仓库目前不是一个可直接编译的固件工程，而是一个硬件开发资料库。开发固件前，建议先阅读 `Airclean_board_Context7_KB.md`，它已经把原始 PDF、Excel 和视觉复核结果整理成适合 AI / Context7 检索的知识库。

## 仓库内容

`Airclean_board_Context7_KB.md` 是当前最重要的入口文件。它汇总了 CMS79F72x MCU、SU-03T 语音模块、空气净化器板级引脚、风扇规格、烧录流程、NTC 温度表和固件实现注意事项，整体置信度为 `0.91`。

`context7.json` 是 Context7 配置文件，用于告诉 Context7 如何解析本仓库、排除哪些临时目录，以及回答时需要遵守哪些边界规则。

`CMS79F72x用户手册_V1.1.pdf` 是 MCU 原始手册，包含引脚、CONFIG、GPIO、ADC、PWM、USART、EEPROM、电气参数和封装信息。

`空气净化器管脚图.xlsx` 是板级引脚分配来源，包含触摸按键、TM1628A 显示驱动、LED、语音 / WiFi UART、风扇、ION 和 TP-401W / 温度传感器连接。

`SU-03T技术开发手册v1.2.pdf` 是离线语音模块资料，包含 SU-03T 引脚、UART 协议、平台配置、语音触发、串口输出和烧录方式。

`烧录说明.pdf` 是 Hummingbird-M / USB 串口升级相关说明。

`风扇规格书.pdf` 是风扇功能规格资料，包含风速档位、定时、预约、ECO 温控和遥控器描述。

`R25=100K±1%   B25-50=3950±1%(250C).xls` 是 100K NTC 阻值温度表和 ADC 参考数据。

## 推荐阅读顺序

先读 `Airclean_board_Context7_KB.md`，快速建立整板认知。

然后读 `空气净化器管脚图.xlsx`，确认实际 MCU 引脚、外设连接和板级命名。

接着读 `CMS79F72x用户手册_V1.1.pdf`，重点看 CONFIG、GPIO、ADC、USART、PWM 和 EEPROM。

如果开发语音功能，再读 `SU-03T技术开发手册v1.2.pdf`，重点确认 `B6/B7` UART1、串口帧格式和烧录流程。

如果开发温控或 ECO 模式，再读 `风扇规格书.pdf` 和 NTC 表，确认温度到风速档位的映射关系。

## 当前关键结论

本板核心 MCU 从已使用引脚看，大概率是 `CMS79F723`，但最终必须以 BOM、芯片丝印或 PCB 封装确认。

四个触摸按键分别是：`RA3/TK3` 定时、`RA0/TK0` 风速、`RA2/TK2` 开关、`RA1/TK1` 滤网重置。

TM1628A 类显示驱动使用 `RA4`、`RA5`、`RA6`，其中 `RA4=DIO`、`RA5=CLK`、`RA6=STB`。

语音模块 / WiFi UART 使用 `RC0` 和 `RC1`。外设 `TX` 接 MCU `RX/RC1`，外设 `RX` 接 MCU `TX/RC0`。

SU-03T 的 UART1 是 `B7=UART1_TXD`、`B6=UART1_RXD`。串口触发默认帧格式为 `AA 55 msgno params 55 AA`，多字节参数低位先发。

风扇 `PWM` 标注在 `RB3`，但 CMS79F72x 手册没有把 `RB3` 列为硬件 CCP/PWM 输出。因此当前固件应先按 GPIO 软件 PWM 设计，不能直接假设 `RB3` 支持硬件 PWM。

TP-401W / 温度传感器数据接 `RB2/AN13`。NTC 参数为 `R25=100KΩ±1%`、`B25/50=3950K±1%`。

NTC 表中可采用的是 `T(℃)`、`Rmin(KΩ)`、`Rmax(KΩ)`、`Rnom(KΩ)`、`Tol.` 以及与 10K 上拉、NTC 接地分压公式一致的 ADC 参考列。`Sheet1` 中部分反向 / 其它参考列存在异常，不应作为温度换算依据。

ECO 模式温度到风速的视觉复核结果为：`<=23°C` 对应 2 档，`24-25°C` 对应 4 档，`26-27°C` 对应 6 档，`28-29°C` 对应 8 档，`30-31°C` 对应 9 档，`32°C` 对应 10 档，`>32°C` 对应 12 档。温度变化需要持续 10 秒后才调整风速。

## Context7 使用方式

本仓库已经提供 `context7.json`。如果把仓库接入 Context7，根目录 Markdown 文件会被解析，`Airclean_board_Context7_KB.md` 会作为主要知识库入口。

建议在 Context7 问答时优先围绕这些主题提问：板级引脚、CMS79F72x 外设配置、SU-03T UART 协议、NTC 温度换算、ECO 风速规则、烧录流程和固件实现边界。

## 固件开发注意事项

不要把 `RB3` 当作硬件 PWM，除非后续原理图或芯片资料能证明它有硬件 PWM 路径。

不要把 SU-03T 文档截图里的 `9600, 8N1` 强行当作所有固件的固定默认值。最终串口参数应以智能公元平台配置和生成固件包为准。

不要只凭当前资料实现 TM1628A 协议。本仓库没有 TM1628A 数据手册，显示 RAM 地址、按键扫描格式和串行时序需要另行确认。

不要只凭本仓库实现遥控器解码。风扇规格书提到编码格式类似 `HT6222`，但没有完整码值表。

EEPROM 适合保存滤网寿命、用户偏好等少量掉电保持状态，不适合高频写入倒计时、实时风速或每次触摸状态。

## 仍需确认

完整原理图、BOM、芯片丝印、TM1628A 数据手册、风扇驱动电路、SU-03T 生成固件中的 UART 代码、WiFi 模块规格、遥控码表、NTC 分压电阻实物值。

这些内容确认后，应同步更新 `Airclean_board_Context7_KB.md`，再重新评估知识库置信度。

## 维护建议

新增资料时，优先把原始 PDF、Excel、图片或固件说明放入仓库根目录或清晰命名的资料目录。

每次新增关键资料后，应更新 `Airclean_board_Context7_KB.md`，并在内容中标明来源、置信度和仍需确认的边界。

临时解析、截图、视觉复核输出不建议提交到仓库，可继续放在 `.codex_tmp/` 这类被 `.gitignore` 排除的目录中。
